Maven partnership

Discovery highlights & Phase 1 implementation direction

We mapped Maven’s recruiter workflows, quantified time drains, and shaped a Phase 1 plan that reduces repeated handling while keeping Mercury + PowerApps stable for day-to-day delivery.

Executive snapshot
  • Strong commercial outcomes today; friction from fragmented tooling and duplicated entry.
  • Documentation, market mapping, and content prep consume ~85–95 hours/week across the working group.
  • Phase 1 keeps Mercury + PowerApps operational while reducing manual handling and setting the foundation for automation and AI.
Phase 1 value targets
  • Structured capture and one-source-of-truth to cut repeated write-ups.
  • Faster routine logging with clearer pathways from raw notes to system entry.
  • Stable, scalable base for later automation/GenAI without disrupting current revenue flows.
Updated wording will follow once the full document is finalized.
Where time goes
  • Documentation, market mapping, and content prep = single largest weekly load.
  • Multi-step conversion of raw call notes into structured fields.
  • Routine duplication between offline notes and PowerApps/Mercury submissions.
Current stack reality
  • Mercury + Dataverse = core ATS/CRM backbone.
  • PowerApps = primary UX for day-to-day logging; effective but still multi-step.
  • ContactOut and other enrichments feed research; Power BI used mainly as static exports for SLT.
  • Shadow Excel files for market mapping/mandates create parallel sources of truth.
Market mapping

Core activities

  • Adding and updating contacts as roles/companies change.
  • Researching up-to-date info and scanning for market movements.
Mandate activities

Core activities

  • Building longlists/shortlists and keeping mandate data current.
  • Consolidating candidates and prepping structured submissions.
Knowledge share

Core activities

  • Capturing and summarizing call/email notes.
  • Searching across notes, Teams, and Engage for reusable insights.
  • Sharing updates in a way that stays discoverable over time.
Phase 1 solution set
Foundational
  • Consolidate shadow IT data (team/individual Excel) into Dataverse with structured ingestion.
  • Configure roles, permissions, and authentication aligned to recruitment workflows.
  • Stand up front-end patterns/components for scalable UI delivery.
Estimated effort: ~4–6 weeks for data consolidation; ~2 weeks for roles/permissions; ~1 week for front-end setup.
Functional
  • Simplify structured capture paths to reduce repeat write-ups and duplicate logging.
  • Streamline mandate data refresh and shortlist maintenance inside Mercury/PowerApps.
  • Improve knowledge visibility with searchable, reusable insights for recruiters and leadership.
Architectural
  • Evaluate architecture choices to keep Mercury + PowerApps stable while enabling future automation/AI workflows.
  • Prioritize scalability, cost control, and data governance in Phase 1 sequencing.
Collaboration with Virtuoso

Coordinate reviews on security, integrations, and architecture at defined build intervals to keep alignment with Maven governance.

Enablement & behaviors
  • Run global training on agreed processes and available functionality.
  • Assign process owners for manuals, SOPs, and beta testing.
  • Recommend a Head of Learning for consistent onboarding and adoption.
Phase 1 roadmap headline

Stabilize the current ecosystem while cutting repeated data handling. Start with structured input improvements, note-conversion refinement, and data consolidation; keep Mercury + PowerApps reliable while preparing for automation and GenAI in later phases.
